# Pen-Time self-hosted updates

This directory mirrors the server root:

```text
/usr/share/nginx/html/pentime-updates/
  app-upgrade-config.json
  pen-time/releases/v1.9.9/
  nginx/pentime-updates.conf
```

Copy the nginx config to:

```bash
cp /usr/share/nginx/html/pentime-updates/nginx/pentime-updates.conf /etc/nginx/conf.d/pentime-updates.conf
nginx -t
systemctl reload nginx
```

For the current app version, upload the build artifacts generated by electron-builder to:

```text
/usr/share/nginx/html/pentime-updates/pen-time/releases/v1.9.9/
```

The release directory must include the generated update metadata, such as `latest.yml`, and the generated installer files. Do not hand-write `latest.yml`; electron-builder must generate it so `sha512`, `path`, `files`, and `version` match the actual installer.

Useful checks:

```bash
curl -I https://pentime-new.com/app-upgrade-config.json
curl -I https://pentime-new.com/pen-time/releases/v1.9.9/latest.yml
curl -I https://www.pentime-new.com/app-upgrade-config.json
```

For a new version:

1. Update `package.json` version.
2. Update `electron-builder.yml` publish URL to the new `/pen-time/releases/vX.Y.Z` directory.
3. Build the installer so electron-builder generates `latest.yml`.
4. Upload the generated installer and update metadata into `pen-time/releases/vX.Y.Z/`.
5. Update `app-upgrade-config.json` so `latest.version` and both feed URLs point at the new version directory.
