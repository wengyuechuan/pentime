# Client Release And Auto Update

Use this for Pen-Time Electron client release tasks.

## Files That Define A Release

- `package.json`: app version, for example `1.10.0`
- `electron-builder.yml`: product name, app id, artifact naming, publish URL, release notes
- `app-upgrade-config.json`: server-side config uploaded to `https://pentime-new.com/app-upgrade-config.json`
- `dist/latest.yml`: generated by Electron Builder; upload it with the installer

Release paths must agree:

```text
package.json version                 -> 1.10.0
electron-builder.yml publish.url     -> https://pentime-new.com/pen-time/releases/v1.10.0
app-upgrade-config.json feedUrls     -> https://pentime-new.com/pen-time/releases/v1.10.0
dist/latest.yml version              -> 1.10.0
```

## Local Backup Before Release

Create a compact recoverable backup of current source changes:

```powershell
$timestamp = Get-Date -Format 'yyyyMMdd-HHmmss'
$backupRoot = 'C:\Users\wengy\code\PenTime-backups'
$backupDir = Join-Path $backupRoot "workingtree-$timestamp"
$zipPath = Join-Path $backupRoot "PenTime-workingtree-$timestamp.zip"
New-Item -ItemType Directory -Path $backupDir -Force | Out-Null
git status --short | Out-File (Join-Path $backupDir 'git-status.txt') -Encoding utf8
git diff --binary | Out-File (Join-Path $backupDir 'tracked-diff.patch') -Encoding utf8
git ls-files --others --exclude-standard | Out-File (Join-Path $backupDir 'untracked-files.txt') -Encoding utf8
Compress-Archive -Path (Join-Path $backupDir '*') -DestinationPath $zipPath -Force
```

Skip `.codex-run/` for commits and normally for backups unless debugging build extraction specifically.

## Build Windows Setup

Stop local dev app before building:

```powershell
Get-CimInstance Win32_Process |
  Where-Object { $_.CommandLine -match 'C:\\Users\\wengy\\code\\PenTime' -and $_.Name -match 'node|electron|powershell' } |
  ForEach-Object { Stop-Process -Id $_.ProcessId -Force }
```

Build x64 Windows setup:

```powershell
pnpm build:win:x64
```

Expected artifacts:

```text
dist/Pen-Time-<version>-x64-setup.exe
dist/Pen-Time-<version>-x64-portable.exe
dist/latest.yml
```

Build includes:

- OpenAPI generation
- node/web/aiCore typecheck
- Electron Vite build
- Electron Builder packaging/signing

Warnings about Vite/Rolldown direct `eval`, dynamic imports, or a temp runtime audit directory cleanup can be non-blocking if the command exits `0`.

## Upload To `pentime-new`

Remote layout:

```text
/usr/share/nginx/html/pentime-updates/
  app-upgrade-config.json
  pen-time/releases/v<version>/
    latest.yml
    Pen-Time-<version>-x64-setup.exe
    Pen-Time-<version>-x64-portable.exe
```

Recommended upload sequence:

1. Create remote release directory.
2. Backup old `app-upgrade-config.json` and Nginx config.
3. Upload installer, portable exe, `latest.yml`, and `app-upgrade-config.json`.
4. Fix ownership and mode.
5. Update `/download/windows` redirect in Nginx.
6. Run `nginx -t` and `systemctl reload nginx`.
7. Verify public URLs.

Use a Python/Paramiko script when `scp` cannot pass passwords non-interactively. Keep the password outside source files.

Remote command pattern:

```bash
VERSION=1.10.0
ROOT=/usr/share/nginx/html/pentime-updates
mkdir -p "$ROOT/pen-time/releases/v$VERSION" "$ROOT/backups/v$VERSION-$(date +%Y%m%d-%H%M%S)"
cp -a "$ROOT/app-upgrade-config.json" "$ROOT/backups/v$VERSION-$(date +%Y%m%d-%H%M%S)/" || true
cp -a /etc/nginx/conf.d/pentime-updates.conf "$ROOT/backups/v$VERSION-$(date +%Y%m%d-%H%M%S)/" || true
chown -R nginx:nginx "$ROOT/pen-time/releases/v$VERSION" "$ROOT/app-upgrade-config.json"
chmod 644 "$ROOT/pen-time/releases/v$VERSION"/* "$ROOT/app-upgrade-config.json"
nginx -t
systemctl reload nginx
```

## Verify Auto Update

Check config:

```powershell
curl.exe -s https://pentime-new.com/app-upgrade-config.json
curl.exe -s https://pentime-new.com/pen-time/releases/v1.10.0/latest.yml
```

Expected `latest.yml`:

```yaml
version: 1.10.0
path: Pen-Time-1.10.0-x64-setup.exe
```

Stable download check:

```powershell
curl.exe -I -L https://pentime-new.com/download/windows
curl.exe -I -L https://www.pentime-new.com/download/windows
```

Expected final `Location` or final URL:

```text
https://pentime-new.com/pen-time/releases/v1.10.0/Pen-Time-1.10.0-x64-setup.exe
```

## GitHub Submit Flow

Before commit:

```powershell
git status --short
git diff --stat
```

Do not add `.codex-run/`.

Commit and push:

```powershell
git add <source files>
git commit --signoff -m "feat: release Pen-Time <version> client"
git -c http.proxy= -c https.proxy= push origin main
```

If hooks format files and block the first commit, run:

```powershell
git add -u
git commit --signoff -m "feat: release Pen-Time <version> client"
```
